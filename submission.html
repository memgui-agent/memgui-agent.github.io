Submission Methods

Method 1: GitHub Pull Request (Recommended)

  1. Fork our GitHub repository
  2. Add your results to data/results.json
  3. Include your agent information and metrics
  4. Create a Pull Request with title: [Leaderboard] Add [Your Agent Name]
  5. Our team will review and merge within 3-5 business days

Method 2: Email Submission

Send your results to: memgui-bench@example.com

Please include:

  • Your agent name and description
  • Results JSON file
  • Paper/code link (if available)
  • Trajectory files (optional, but recommended for verification)

Trajectory Data

Important: Trajectory Submission

Trajectory submission is optional but highly recommended for the following reasons:

  • Enables verification of your results by our team
  • Allows reproduction and analysis by other researchers
  • Contributes to the research community's understanding of agent behavior

Trajectory Storage Format

Each task execution should be saved as a JSON file with the following structure:

{
  "task_id": "task_001",
  "agent_name": "YourAgent",
  "attempt": 1,
  "success": true,
  "steps": [
    {
      "step_id": 1,
      "timestamp": "2026-01-15T10:30:00Z",
      "screenshot_before": "screenshots/task_001_step_001_before.png",
      "screenshot_after": "screenshots/task_001_step_001_after.png",
      "action": {
        "type": "CLICK",
        "coordinates": [540, 1200],
        "element_description": "Settings button"
      },
      "agent_reasoning": "Need to access settings to change preferences",
      "ui_hierarchy": "ui_trees/task_001_step_001.json"
    }
  ],
  "total_steps": 15,
  "total_time_seconds": 120.5,
  "evaluation_result": {
    "success": true,
    "irr": 0.85,
    "stage_passed": "stage_1"
  }
}

Sharing Trajectories

You can share trajectories through:

  • HuggingFace Datasets: Upload to HuggingFace Hub and provide the dataset link
  • Google Drive / OneDrive: Share a compressed archive with us
  • GitHub Release: Attach as release assets in your agent's repository

Results Format

Your submission should include metrics for both short-term and long-term memory evaluation:

{
  "agent_name": "YourAgent",
  "agent_description": "Brief description of your agent",

  "paper_link": "https://arxiv.org/abs/...",
  "code_link": "https://github.com/...",
  "trajectory_link": "https://huggingface.co/datasets/...",
  
  "short_term": {
    "easy": 41.7,
    "medium": 19.0,
    "hard": 18.4,
    "overall": 27.3,
    "irr": 39.5,
    "mtpr": 0.45,

  "long_term": {

Review Process

Submission Received

We acknowledge your submission within 24 hours.

Format Validation

We verify that all required fields are present and correctly formatted.

Results Verification

If trajectories are provided, we run spot-checks using MemGUI-Eval.

Leaderboard Update

Approved results are added to the leaderboard within 3-5 business days.

Frequently Asked Questions

Trajectory submission is optional but recommended. While we accept results without trajectories, providing them enables verification and helps the research community. Submissions with trajectories may receive priority in the review process.

Yes! You can submit updated results at any time. We maintain version history and will update the leaderboard with your latest results. Please indicate in your submission that this is an update to existing results.

Minimum required metrics:
  • Short-term: SR (Easy, Medium, Hard, Overall)
  • Long-term: SR@3 (Easy, Medium, Hard, Overall)
Recommended additional metrics:
  • IRR, MTPR (for short-term)
  • FRR, Improvement (for long-term)
  • Time/Step, Cost/Step (for efficiency)

We perform the following verification steps:
  1. Check format compliance and metric calculations
  2. If trajectories are provided, run MemGUI-Eval on a random sample (10-20% of tasks)
  3. Compare against known baselines for sanity checks
  4. For significant improvements, we may request additional evidence or reproduction instructions
