MemGUI-Bench

Benchmarking Memory of Mobile GUI Agents in Dynamic Environments

Paper Code Dataset Leaderboard

Overview

Current mobile GUI agent benchmarks systematically fail to assess memory capabilities, with only 5.2-11.8% memory-related tasks and no cross-session learning evaluation.

MemGUI-Bench introduces a comprehensive memory-centric benchmark with pass@k evaluation and staged LLM-as-judge evaluation, designed to rigorously test agents' ability to retain and utilize information across complex, multi-application workflows.

Evaluation Metrics

♣ Short-Term Memory (pass@1)

  • SR - Success Rate: Baseline performance measurement
  • IRR - Information Retention Rate: Memory fidelity metric
  • MTPR - Memory-Task Proficiency Ratio: Memory-specific capability isolation

♠ Long-Term Memory (pass@k)

  • SR@k - Multi-Attempt Success Rate: Cross-session learning capability
  • FRR - Failure Recovery Rate: Learning from failure efficiency

Framework Overview

Figure 1: MemGUI-Bench Overview

First comprehensive benchmark for GUI agent memory evaluation, featuring memory taxonomy, task suite, and specialized metrics.

Figure 2: MemGUI-Eval Progressive Scrutiny Pipeline

Three-stage evaluation: Cost-effective triage → Full semantic analysis → Targeted visual verification.

Figure 3: Task Suite Statistics

68 real-world scenarios, 26 applications, 3 difficulty levels, and 4 app-crossing spans.

Figure 4: Plug-and-Play Framework

Snapshot-based architecture with parallel execution, automatic evaluation, and retry mechanisms.

4-10× Capability Gap

Standard benchmarks systematically overestimate agent capabilities. Performance drops dramatically from AndroidWorld to MemGUI-Bench.

🧠

Memory is Essential

Short-term memory is mandatory (removing it causes SR to collapse). Long-term memory enables +21.9pp improvement.

📱

Cross-App Bottleneck

16-40pp performance degradation from 1-app to 4-app tasks, revealing cross-application transfer as the primary challenge.

⚡

Context Length Matters

Long-context capability yields +18.8pp improvement, demonstrating untapped potential in current architectures.
